## Scope (AFK defaults locked)

### In scope (this sub-phase = 6.4.A)

1. **XAML scanner regression test** — scans `src/UI/AuraCore.UI.Avalonia/Views/**/*.axaml` for hardcoded `Text=`/`Content=`/`ToolTip=`/`Watermark=`/`Header=`/`Title=` attributes holding user-facing text.
2. **Known-offender ratchet file** — JSON or in-test list of file paths currently allowed to have hardcoded strings. Test fails if a NEW file gains hardcoded strings OR a grandfathered file's offenders drop to zero (forcing removal from list).
3. **Whitelist of legitimate literals** — scanner ignores single-char, pure-acronym (all-caps 1-5 chars), pure-punctuation (`—`, `•`, `→`, `·`), and pure-numeric strings.
4. **Reduce offender count on 2-3 high-value modules** — SidebarView, SettingsView, and one or two more based on bang-for-buck assessment after scanner baseline lands. Each fixed module: add new EN+TR keys to `LocalizationService`, swap XAML literals for `{x:Static loc:S.Current[key]}` or equivalent binding pattern (mirror existing localized module pattern).
5. **LanguageChanged subscription** — if a module currently doesn't re-render on language switch, add the subscription pattern (existing `LocalizationService.LanguageChanged` event).

### Out of scope (deferred for subsequent sessions)

- Code-behind string-literal scanner. XAML is the bigger surface + simpler to parse. Code-behind scanner follows in a future sub-phase.
- Fixing all remaining modules. Sweep is incremental — each future session removes files from the offender list.
- Adding a 3rd language. Explicitly forbidden by the roadmap lock until TR coverage reaches 100%.
- Dialog-title scanner. Covered transitively by the generic attribute scan; dialog-specific follow-up deferred.

## Architecture

### Scanner test shape

Location: `tests/AuraCore.Tests.UI.Avalonia/Localization/HardcodedStringScannerTests.cs`

```csharp
[Fact]
public void No_new_files_with_hardcoded_UI_strings()
{
    var offenderFiles = ScanViewsForHardcodedStrings();
    var expected = GrandfatheredFiles;

    var newOffenders = offenderFiles.Except(expected).ToList();
    var cleanedUp = expected.Except(offenderFiles).ToList();

    Assert.Empty(newOffenders); // Added literals in a previously-clean file
    Assert.Empty(cleanedUp);    // A file got fixed — remove from GrandfatheredFiles
}
```

**Whitelist rules** (string IS considered legitimate and ignored):
- Length ≤ 1 (single char like "A", "—")
- All-uppercase and length ≤ 5 (acronyms like "SMART", "CPU", "RAM", "GPU", "OS")
- Pure non-alphanumeric (`—`, `·`, `•`, `→`, `…`)
- Pure numeric (`0`, `100`, `3.14`)
- Empty or all-whitespace

**Detection rules** (attribute value IS flagged as offender):
- Starts with a letter or digit (not `{`)
- Contains at least one letter
- Not on whitelist

**Scanned attributes:** `Text`, `Content`, `ToolTip`, `ToolTip.Tip`, `Watermark`, `Header`, `Title`, `Label`.

**File scope:** `src/UI/AuraCore.UI.Avalonia/Views/**/*.axaml`. Excludes `Dev/ComponentGalleryWindow.axaml` (developer tool, not user-facing).

### Known-offenders data

Stored as a hard-coded list at the top of `HardcodedStringScannerTests.cs` — not a separate JSON file. Makes the ratchet state visible in git diffs when modules get cleaned up.

Format: relative file paths from repo root, e.g. `src/UI/AuraCore.UI.Avalonia/Views/Pages/DashboardView.axaml`.

Initial population: whatever the first scanner run reports.

### Per-module fix workflow

1. Open the module's `.axaml`, grep for `Text="[^{]`, `Content="[^{]`, etc.
2. For each offender, choose (or create) a localization key in `LocalizationService.cs` — add entries to BOTH `EN` and `TR` dicts.
3. Replace the XAML literal with a binding. Mirror existing localized patterns in the codebase. Example:
   ```xml
   <!-- Before -->
   <TextBlock Text="Quick Actions"/>

   <!-- After -->
   <TextBlock Text="{Binding Source={x:Static loc:LocalizationService.Current}, Path=[dash.quickActions]}"/>
   ```
   Or the code-behind assigns via `Get(...)` on `Loaded` + `LanguageChanged`.
4. If the module lacked `LanguageChanged` subscription, add one in code-behind. Refresh localized strings on event.
5. Remove the file from the grandfather list in the scanner test.
6. Run the scanner test — it should pass.
7. Commit.

The existing pattern in `SettingsView.axaml.cs` (post-Phase 6.3) is the template: code-behind sets `.Text` from `LocalizationService.Get(...)` in a dedicated refresh method, wires `LanguageChanged` to call that method.

## Risk

1. **Binding pattern mismatch.** The existing codebase mixes XAML bindings (`{Binding ...[key]}`) and code-behind assignments (`ctrl.Text = LocalizationService.Get("...")`). When fixing a module, mirror what that file already does; don't introduce a third pattern.
2. **Existing false positives.** Some XAML may have legitimate-looking strings that ARE user-facing but short enough to match the whitelist (e.g., "OK", "Cancel"). Those are still offenders — but filtering them out is fine for the initial gate. A future sweep can tighten the whitelist.
3. **AvaloniaTestApplication bootstrap.** The scanner test is not a render test — it's a file-system + parsing test. It doesn't need `[AvaloniaFact]`; a plain `[Fact]` suffices.
4. **XAML namespace prefixes.** The scanner must handle `Text=` but also `ns:Text=` (if a view uses a custom namespace for attributes — unlikely but possible). Start with simple attribute match; escalate if false negatives appear.
