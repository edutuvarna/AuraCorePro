## 3. `IAuraCoreLLM.ReloadAsync` — locked shape (from brainstorm)

```csharp
namespace AuraCore.Application.Interfaces.Platform;   // or wherever IAuraCoreLLM lives today

public interface IAuraCoreLLM : INotifyPropertyChanged
{
    // ... existing members preserved ...

    /// <summary>
    /// Reloads the LLM without app restart. Null <paramref name="newConfig"/>
    /// reloads the current config from disk (useful when the user updated the
    /// model file on disk). Non-null hot-swaps to the new config.
    /// </summary>
    /// <remarks>
    /// - Throws <see cref="InvalidOperationException"/> if <see cref="IsReloading"/>
    ///   is already true (concurrent-reload guard).
    /// - If an inference is in flight, the reload awaits its completion before
    ///   disposing the old instance.
    /// - Sets <see cref="IsReloading"/> true at entry, false on exit (including
    ///   exceptions). INPC raised for both transitions.
    /// </remarks>
    Task ReloadAsync(LlmConfiguration? newConfig = null, CancellationToken ct = default);

    /// <summary>
    /// True while <see cref="ReloadAsync"/> is executing. INPC-raised for UI
    /// state (disable inference buttons, show spinner).
    /// </summary>
    bool IsReloading { get; }
}
```

Test matrix (unit, 6 tests):
1. Happy path: `ReloadAsync()` with null reloads same config; IsReloading toggles true→false; instance swap verified via reference equality check on internal state.
2. Config-swap path: `ReloadAsync(newConfig)` disposes old instance, instantiates new one with newConfig, invocation count on factory increments.
3. IsReloading INPC: subscribe to PropertyChanged, invoke ReloadAsync, assert exactly 2 `IsReloading` changes (false→true, true→false).
4. Concurrent-call rejection: invoke ReloadAsync, don't await, immediately invoke again → second call throws `InvalidOperationException` with message containing "already reloading".
5. Inference-in-flight drain: start a long inference task, invoke ReloadAsync mid-inference, assert reload awaits inference completion before proceeding.
6. Cancellation mid-reload: pass a CT, cancel before disposal step → IsReloading resets to false, swap aborted, old instance still operational.

## 4. HSTS Nginx live-SSH ops plan (item 6)

Production target: `auracore.pro` origin server at `164.92.228.183` (memory: `reference_admin_panel_auth.md` + `project_nanoclaw_kali_droplet.md`).

Current state (per Phase 3 design doc §6): origin Nginx sends `Strict-Transport-Security: max-age=31536000; includeSubDomains; preload`. The `preload` token is problematic only IF submitted to `hstspreload.org` — today it isn't submitted, so the token is inert. But keeping it is risky: any accidental submission locks the domain into the Chrome preload list for months.

**Fix:** drop the `preload` token from the Strict-Transport-Security directive. Keep `max-age=31536000` (1 year) and `includeSubDomains`.

**SSH procedure at execution time:**

1. SSH to `164.92.228.183` (user-provided credentials — ask at dispatch, don't assume).
2. Locate Nginx site config: `sudo grep -rn "Strict-Transport-Security" /etc/nginx/`.
3. Backup original: `sudo cp <site.conf> <site.conf>.bak.5.4.phase`.
4. Edit directive in-place: remove `; preload` suffix only. Keep `always` flag if present.
5. Test: `sudo nginx -t` (must pass; abort if syntax error).
6. Graceful reload: `sudo nginx -s reload` (zero-drop; workers finish in-flight requests, then pick up new config).
7. Verify: `curl -sI https://auracore.pro | grep -i strict-transport-security` → must NOT contain `preload`.
8. Cross-check `admin.auracore.pro`: same verification.
9. Rollback-if-needed: `sudo cp <site.conf>.bak.5.4.phase <site.conf> && sudo nginx -s reload`.

Document the full SSH session transcript in `docs/deploy/2026-04-17-hsts-preload-cleanup.md` for audit trail (what ran, what output, before/after headers).

## 5. `INavigationService` contract (item 4)

```csharp
namespace AuraCore.Application.Interfaces.Platform;

public interface INavigationService
{
    /// <summary>
    /// Requests navigation to the given sectionId. Current well-known ids:
    /// "ai-recommendations", "ai-insights", "ai-schedule" (sub-sections of
    /// AIFeaturesView). Additional module ids may be added without interface
    /// change. Consumers (MainWindow) subscribe to <see cref="SectionRequested"/>.
    /// </summary>
    void NavigateTo(string sectionId);

    /// <summary>Raised when <see cref="NavigateTo"/> is called. MainWindow
    /// routes by inspecting sectionId + switches pages / scrolls accordingly.</summary>
    event EventHandler<NavigationRequestedEventArgs>? SectionRequested;
}

public sealed class NavigationRequestedEventArgs : EventArgs
{
    public string SectionId { get; }
    public NavigationRequestedEventArgs(string sectionId) => SectionId = sectionId;
}
```

Default impl `NavigationService` in `AuraCore.Desktop.Services/Navigation/NavigationService.cs` — singleton, trivial ctor.

MainWindow code-behind subscribes to `SectionRequested` in ctor (same pattern as 5.3 `BoundsObserver`), in handler:
- If sectionId starts with `"ai-"`: ensure AIFeaturesView is the active page, call `AIFeaturesView.ShowSection(sectionId.Substring("ai-".Length))` (new public method — swap child section container).
- Else: log warn, no-op (graceful forward-compat).

Dashboard's Smart Optimize button click handler:
```csharp
private void OnSmartOptimizeClicked(object? sender, RoutedEventArgs e)
{
    var nav = App.Services?.GetService<INavigationService>();
    nav?.NavigateTo("ai-recommendations");
}
```

`AIFeaturesView.ShowSection(string subSectionId)` — public method that tab-switches the inner content (existing section enum pattern from Phase 3). Tests verify the method exists + the Dashboard→Nav→AIFeaturesView round-trip via mocked `INavigationService`.

## 10. Security posture

- Zero new third-party NuGet packages — all additions are net-new source files in existing csprojs.
- `INavigationService.NavigateTo(string)` — string is internal-only (from app code), no external input reaches it.
- `UppercaseTransformConverter` — pure string transform, no injection surface.
- `IAuraCoreLLM.ReloadAsync` — model file is already on the user's local filesystem; reload doesn't open any new attack surface. The inference-drain guard prevents race conditions that could corrupt model state.
- HSTS ops — live production change; mitigated by the test-before-reload discipline + backup + rollback plan in §4.
