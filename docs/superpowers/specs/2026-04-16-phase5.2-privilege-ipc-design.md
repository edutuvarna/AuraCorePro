## 1. Purpose

Ship the cross-cutting privilege IPC infrastructure that Phase 5.2 of the umbrella roadmap blocks on. Three outputs:

1. **`IShellCommandService` abstraction landed** (contract frozen in umbrella §4.6, implementations + DI wiring are this sub-wave's work).
2. **Linux real privilege path**: `auracore-privhelper` .NET 8 daemon + polkit policy + D-Bus activation + lazy installer; 6 Linux engine migrations.
3. **macOS real privilege path**: `pro.auracore.privhelper` XPC helper tool bundled via `SMAppService`; 5 macOS engine migrations.

Windows remains on existing empty-shell `AuraCore.PrivilegedService` scaffold this wave (fleshed out in Phase 5.5 with the Driver/Defender/Service write-capability work).

## 4. Contract surfaces (locked)

### 4.1 `IShellCommandService` — from umbrella §4.6, copied here for implementation reference

```csharp
namespace AuraCore.Application.Interfaces.Platform;

public interface IShellCommandService
{
    Task<ShellResult> RunPrivilegedAsync(PrivilegedCommand command, CancellationToken ct = default);
}

public sealed record PrivilegedCommand(
    string Id,                      // whitelist action id — maps to polkit action / XPC method / Windows IPC op
    string Executable,
    string[] Arguments,
    int TimeoutSeconds = 60);

public sealed record ShellResult(
    bool Success,
    int ExitCode,
    string Stdout,
    string Stderr,
    PrivilegeAuthResult AuthResult);

public enum PrivilegeAuthResult
{
    AlreadyAuthorized,   // op ran; auth was cached / not needed
    Prompted,            // op ran after user approved auth prompt
    Denied,              // user explicitly refused auth / polkit denied
    HelperMissing        // helper not installed on system
}
```

## 6. Engine migration pattern (copied from umbrella §4.7)

```csharp
// Before (existing)
var r = await ProcessRunner.RunAsync("sudo", "-n journalctl --vacuum-size=500M", ct);

// After (5.2 target)
var r = await _shellCommandService.RunPrivilegedAsync(
    new PrivilegedCommand(
        Id: "journal",                                         // action id whitelisted in daemon
        Executable: "journalctl",
        Arguments: new[] { "--vacuum-size=500M" },
        TimeoutSeconds: 60),
    ct);

if (!r.Success)
{
    // One of Denied / HelperMissing / runtime failure
    // VM layer observes PrivilegeAuthResult and surfaces module-local error + optional global banner
    return EngineOutcome.Failed(r);
}
```

VM/view layer consumes `ShellResult.AuthResult` to route to the right UX:
- `Denied` → module-local error card with retry
- `HelperMissing` → global banner (via `IHelperAvailabilityService.Missing = true`)
- `Success=false` with `AuthResult=AlreadyAuthorized|Prompted` → genuine op failure; module-local error with stderr detail

## 7. UX flows

### 7.1 First-use Linux install

1. User clicks privileged action (e.g., Journal → Vacuum 500MB).
2. Engine calls `IShellCommandService.RunPrivilegedAsync`.
3. `LinuxShellCommandService` checks: is daemon reachable on system bus? No.
4. Returns `ShellResult(Success=false, AuthResult=HelperMissing)`.
5. VM catches → shows `PrivilegeHelperInstallDialog` (not banner — user is actively invoking; modal is appropriate).
6. Dialog body: "AuraCorePro needs to install a small helper service (`auracore-privhelper`) to run privileged system commands on your behalf. You'll be asked for your admin password. [Install] [Cancel]"
7. User clicks Install → `PrivHelperInstaller.RunPkexecInstallAsync()` → OS `pkexec` password dialog.
8. Success → dialog closes, engine auto-retries original action → normal polkit prompt on first privileged op → success.
9. Failure/Cancel → dialog closes, module-local error toast.

### 7.2 First-use macOS install

Same shape, but step 7 uses `SMAppService.daemon(plistName:"pro.auracore.privhelper.plist").register()` which triggers macOS's native "AuraCorePro wants to install a helper tool" dialog + admin password prompt. On dev builds without proper signing, dialog step skipped; `InProcessShellCommandService` used (direct command, no elevation — warning logged).

### 7.3 Subsequent uses

Helper present + authorized within 5-minute cache → `AuthResult=AlreadyAuthorized`, no prompts.
Helper present + cache expired → native polkit/XPC prompt → `AuthResult=Prompted`.
Helper present + user denies prompt → `AuthResult=Denied` → module-local error with "Retry" affordance.

### 7.4 HelperMissing global banner

Triggered when any `RunPrivilegedAsync` returns `AuthResult=HelperMissing` and user hasn't dismissed this session.
Copy: "Privilege helper is not installed. Some modules can't perform system changes. [Install now] [Dismiss]"
Persists at top of main window until installed or user dismisses. Dismissing shows a settings toggle to re-enable.

## 9. Security considerations

- **Argv validation in daemon is the primary defense.** polkit prompt gates access to daemon; daemon's whitelist decides what argv patterns are accepted per action. E.g., `journal` action validator:
  ```csharp
  if (!argv.All(arg => Regex.IsMatch(arg, @"^--vacuum-(size|time)=\d+[MGkmhd]$|^--rotate$")))
      return Reject("invalid journal argv");
  ```
- **No shell interpolation.** All commands invoked via `ProcessStartInfo.ArgumentList`; no `bash -c`.
- **PrivilegedCommand.Id is untrusted from engine.** Daemon maps id → handler, not engine-supplied path. Engine can't invoke arbitrary `Executable`.
- **Ignore `PrivilegedCommand.Executable` server-side.** Daemon uses hard-coded executable paths per action id. The field is a client-side hint / documentation aid only; daemon overrides.
- **systemd unit runs as root.** Required for the operations (journalctl vacuum, apt-get, docker, kernel pkg removal, grub). Unit file includes `ProtectSystem=strict`, `ProtectHome=true`, `PrivateTmp=true`, `NoNewPrivileges=false` (must remain false because apt-get fork/execs), `CapabilityBoundingSet=CAP_DAC_OVERRIDE CAP_SYS_ADMIN` (minimal required — refined in plan).
- **macOS helper runs as root.** `launchd` plist specifies `UserName=root`. Entitlements restrict what XPC messages are accepted — sender must match main app's team ID + bundle ID.
- **Audit every op to journald / os_log.** Format: `[privhelper] action=<id> argv=<joined> uid=<caller> exit=<code> duration_ms=<n>`.
