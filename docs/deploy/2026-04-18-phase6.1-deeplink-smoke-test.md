# Phase 6.1 Deep-link Smoke Test

**Date:** 2026-04-18  
**Feature:** `auracore://` URL scheme routing

## Prereqs

- Debug build of `AuraCore.UI.Avalonia.exe` at:
  `src/UI/AuraCore.UI.Avalonia/bin/Debug/net8.0/AuraCore.UI.Avalonia.exe`
- Windows host (scheme registration is Windows-only in Phase 6.1)

Build first if needed:
```bash
cd C:/Users/Admin/Desktop/AuraCorePro/AuraCorePro
dotnet build src/UI/AuraCore.UI.Avalonia/AuraCore.UI.Avalonia.csproj -c Debug --nologo
```

## Scenarios

### 1. Auto-registration on first launch

1. Ensure no prior registration: `Remove-Item HKCU:\Software\Classes\auracore -Recurse -Force -ErrorAction SilentlyContinue`
2. Launch AuraCore: `Start-Process "src\UI\AuraCore.UI.Avalonia\bin\Debug\net8.0\AuraCore.UI.Avalonia.exe"`
3. Wait ~3 seconds for app to load.
4. Verify registry: `Get-ItemProperty HKCU:\Software\Classes\auracore\shell\open\command | Format-List`
5. Expected: the `(Default)` value should be `"<full-path-to-AuraCore.UI.Avalonia.exe>" "%1"`

### 2. Deep-link opens at Disk Health (app already running)

1. With AuraCore running from scenario 1, execute: `Start-Process "auracore://disk-health"`
2. Expected:
   - AuraCore window comes to foreground
   - Navigation lands on Disk Health view
   - No new AuraCore process is spawned (check via Task Manager)

### 3. Deep-link from cold start

1. Close all AuraCore instances.
2. Execute: `Start-Process "auracore://disk-health"`
3. Expected: AuraCore launches and opens directly at Disk Health (not Dashboard).

### 4. Deep-link to a module

1. With AuraCore running, execute: `Start-Process "auracore://module/driver-updater"`
2. Expected: AuraCore focuses and navigates to Driver Updater module page.

### 5. Unknown ID → silent fallback to default view

1. With AuraCore running, execute: `Start-Process "auracore://not-a-real-section"`
2. Expected:
   - AuraCore window focuses
   - Current view remains unchanged (no crash, no error dialog)

### 6. Malicious input ignored

1. Execute: `Start-Process "auracore://../../system32"`
2. Expected:
   - AuraCore window focuses (or opens cold)
   - No filesystem access, no error dialog, no crash
   - Current view unchanged (or lands on Dashboard for cold start)

### 7. Opt-out via registry DWORD

1. Set opt-out flag: `New-ItemProperty -Path HKCU:\Software\AuraCorePro -Name "DisableUrlSchemeAutoRegister" -Value 1 -PropertyType DWORD -Force`
2. Remove scheme: `Remove-Item HKCU:\Software\Classes\auracore -Recurse -Force -ErrorAction SilentlyContinue`
3. Launch AuraCore.
4. Verify scheme NOT re-registered: `Test-Path HKCU:\Software\Classes\auracore` should return `False`
5. Cleanup: `Remove-ItemProperty HKCU:\Software\AuraCorePro -Name "DisableUrlSchemeAutoRegister"`

### 8. Two concurrent launches

1. With AuraCore running, execute twice in rapid succession:
   ```powershell
   Start-Process "auracore://ai-insights"
   Start-Process "auracore://disk-health"
   ```
2. Expected:
   - Still only one AuraCore process (verify Task Manager)
   - App navigates to whichever URL arrived later (ordering unspecified but no duplicates)

## Uninstall scheme manually

```powershell
Remove-Item HKCU:\Software\Classes\auracore -Recurse -Force
```

Or use the uninstall script:

```bash
powershell -File scripts/uninstall-url-scheme.ps1
```

## Expected behavior in scripts/install-url-scheme.ps1 (manual registration)

Run `powershell -File scripts/install-url-scheme.ps1` to register without launching the app:

```powershell
# Dry-run mode (no changes)
powershell -File scripts/install-url-scheme.ps1 -WhatIf

# Register for a specific path
powershell -File scripts/install-url-scheme.ps1 -BinaryPath "C:\custom\AuraCore.exe"
```

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Link does nothing | Scheme not registered | Run `install-url-scheme.ps1` |
| Link launches wrong binary | Registered path drifted | Run `install-url-scheme.ps1 -BinaryPath <correct>` |
| Second launch spawns duplicate window | Named Pipe gateway not running | Check app started as primary (not killed); restart |
| "Your Windows blocked this action" popup | SmartScreen on unsigned exe | Expected pre-ship-prep; bypass for dev testing |
