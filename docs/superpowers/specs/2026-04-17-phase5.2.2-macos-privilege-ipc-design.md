## 3. Decisions (defaulted-recommended under AFK mode)

### 3.1 XPC transport model

**Choice: Mach service via `libxpc` C API.** Daemon claims `pro.auracore.PrivHelper` at startup via `xpc_connection_create_mach_service` (listener role). `launchd` plist's `MachServices` dict maps that name to the daemon. Client calls `xpc_connection_create_mach_service` with the name.

Rejected alternatives:
- Anonymous endpoint passed via SMAppService ‚Äî more complex lifecycle for no ergonomic win.
- `NSXPCConnection` via Cocoa ‚Äî requires Xamarin.Mac (large dep, CVE-history-heavy) or ObjC runtime wrappers (extra complexity). Task 25 deliberately avoided that ecosystem.

### 3.2 XPC payload schema

**Choice: `xpc_dictionary` with primitive types only.**

Request wire format:
```
{
    "action_id":       string       // e.g. "dns-flush", "purgeable"
    "args":            [string...]  // argv tokens, whitelisted by validator
    "timeout_seconds": int64        // per-call timeout for spawned process
}
```

Reply wire format:
```
{
    "exit_code":  int64
    "stdout":     string
    "stderr":     string
    "auth_state": string   // "cached" | "rejected"  (see ¬ß3.5)
}
```

**Strict key allowlist.** Any incoming dictionary containing extra keys is rejected with `auth_state="rejected"`, `exit_code=-101`. Deliberately NO custom codable objects. Prevents deserialization gadget chains ‚Äî `xpc_dictionary` + `xpc_array` are built-in XPC types with no user-level class graph to reconstruct.

Rejected alternatives:
- Protobuf/FlatBuffers ‚Äî heavy dep, CVE history, overkill for 3‚Äì4 fields.
- JSON string in a single `data` blob ‚Äî deserialization-attack surface via custom parser.

### 3.3 Peer identity verification

**Choice: Per-call `SecCodeCheckValidity` with a strict `SecRequirement`.**

Flow in the daemon's XPC handler (Task 26):

1. Get the calling pid via `xpc_connection_get_pid(connection)`.
2. `SecCodeCopyGuestWithAttributes(host=NULL, attributes={pid:<pid>}, flags=0, outGuest=&code)` ‚Äî obtain a `SecCodeRef` for the caller.
3. `SecRequirementCreateWithString(CFString("anchor apple generic and identifier \"pro.auracore.auracorepro\" and certificate leaf[subject.OU] = \"<TEAM_ID>\""), flags=0, outRequirement=&req)`.
4. `SecCodeCheckValidity(code, flags=kSecCSStrictValidate|kSecCSCheckAllArchitectures, req)` ‚Äî returns 0 (errSecSuccess) iff caller matches.
5. If non-zero, reject the XPC call with `auth_state="rejected"` and log a `peer_identity_mismatch` event.

**Not just UID match.** `xpc_connection_get_euid == caller_uid` is insufficient: any process running as the user could impersonate.

### 3.4 TEAM_ID parameterization

**Choice: Compile-time placeholder, signing-time substitution.**

`src/Service/AuraCore.PrivHelper.MacOS/SecurityConfig.cs`:
```csharp
internal static class SecurityConfig
{
    // Replaced at signing time by the build pipeline's `sed -i`.
    // Dev builds ship the placeholder; PeerVerifier treats placeholder
    // as "soft-fail" (log warning, continue) in DEBUG, "hard-fail"
    // (reject XPC call) in RELEASE.
    internal const string ExpectedTeamId = "AURACORE_TEAM_ID_PLACEHOLDER";
    internal const string ExpectedBundleId = "pro.auracore.auracorepro";
}
```

The signing-pipeline task (ship-prep, out of Phase 5.2 scope) does:
```bash
sed -i '' "s/AURACORE_TEAM_ID_PLACEHOLDER/$TEAM_ID/g" \
    AuraCorePro.app/Contents/Library/LaunchDaemons/pro.auracore.privhelper.exe.dSYM/Contents/Resources/DWARF/...
# (or rebuilds the helper with -p:AuraCoreTeamId=$TEAM_ID ‚Üí csproj reads it via Choose/When)
```

Rejected alternatives:
- Env var read at daemon startup ‚Äî root-owned env vars are also attack surface.
- Signed config file ‚Äî another verification layer to get right.
- Hardcoded literal ‚Äî painful to update if team ID changes.

### 3.5 AuthState source (macOS has no polkit)

**Choice: Daemon never distinguishes "cached" from "prompted"; always returns `"cached"` on success.**

Rationale: macOS privileged helpers, once installed via `SMAppService.daemon(plistName:).register()`, run as root under launchd. The user is prompted ONCE at registration time (the macOS "allow AuraCorePro helper" dialog). After that, the helper is always authorized. There is no per-call re-prompt unless we opt into `LocalAuthentication` (TouchID / password), which is out of scope for Phase 5.2.2 MVP.

Mapping on client side (Task 29 `MacOSShellCommandService`):
- `"cached"` ‚Üí `PrivilegeAuthResult.AlreadyAuthorized`
- `"rejected"` ‚Üí `PrivilegeAuthResult.HelperMissing` (whitelist rejection = helper can't serve this op; treat as install-resettable)
- XPC connection failure ‚Üí `HelperMissing`
- XPC call throws ‚Üí `HelperMissing` + `ExitCode = -4`

The `Prompted` enum value exists for cross-platform parity but macOS never emits it.

### 3.6 Notarization + codesign strategy

**Choice: Ship-prep concern; 5.2.2 scope is "buildable + documented".**

5.2.2 ships:
- Docs in spec appendix A (this file ¬ß7) covering the 3 codesign + 1 notarytool commands.
- Build discipline: `codesign --force --options runtime --entitlements entitlements.plist --sign "$CERT_ID" ...` in ship-prep pipeline.
- Dev builds: NO codesign. `MacOSShellCommandService` detects unsigned + `Debugger.IsAttached` ‚Üí falls back to `InProcessShellCommandService` with a warning log.

The pipeline automation itself is Phase 5.2.5 (or later) ship-prep wave ‚Äî NOT 5.2.2.

### 3.7 Apple Silicon support

**Choice: Universal binary via two publish commands + `lipo -create`.**

Ship-prep pipeline glues this; 5.2.2 code itself is portable.

### 3.9 Action IDs + validators

**Choice: One action ID per migrating module. 4 clean IDs + strict per-action validators.**

| Action ID | Executable (daemon-hardcoded) | Argv pattern (allowed) |
|---|---|---|
| `dns-flush` | `/usr/bin/dscacheutil` | `["-flushcache"]` OR `[]` (no args also OK; daemon treats as `-flushcache`) |
| `purgeable` | `/usr/bin/tmutil` | First arg MUST be `thinlocalsnapshots`; second arg MUST be `/` (root mount only); third arg numeric `^\d{1,10}$` (bytes); fourth arg numeric `^[1-4]$` (urgency) |
| `spotlight` | `/usr/bin/mdutil` | First arg MUST be `-a` or a safe path; second arg in `-i on\|-i off\|-E`; rejects absolute paths outside `/Users/` and `/Volumes/` |
| `time-machine` | `/usr/bin/tmutil` | First arg in `startbackup\|stopbackup\|listbackups\|deletelocalsnapshots\|enable\|disable`; flags `--auto\|--no-auto` only |

If Task 31 decides to migrate SwapOptimizer, add 5th action ID `swap` pointing to `/usr/sbin/sysctl` with argv whitelist on specific `vm.*` keys.

### 3.10 SIP-protected-path defense

**Choice: Validator rejects paths matching SIP-protected prefixes BEFORE dispatching to the daemon.**

Daemon-side `ShellMeta.IsSipProtectedPath(path)` returns `true` for any path starting with:
- `/System/` (all)
- `/bin/`, `/sbin/`, `/usr/bin/`, `/usr/sbin/`
- `/usr/libexec/`, `/usr/lib/`
- `/private/var/db/` (dslocal, sudoers, etc.)
- `/.vol/`, `/.DocumentRevisions-V100/`

Exceptions: `/usr/local/` is NOT SIP-protected; user data in `/Users/` / `/Volumes/` is not either.

Validators (¬ß3.9) reject SIP-protected destination paths with `auth_state="rejected"`. Defense-in-depth ‚Äî the kernel would block writes anyway, but surfacing the rejection at validation time yields a clean error instead of a kernel EACCES.

### 3.11 Audit logging target

**Choice: `os_log` via P/Invoke, with stderr fallback.**

`src/Service/AuraCore.PrivHelper.MacOS/Interop/OsLog.cs` (new in Task 26):

```csharp
internal static partial class OsLog
{
    private const string Library = "/usr/lib/system/libsystem_trace.dylib";

    [LibraryImport(Library)]
    internal static partial IntPtr os_log_create(
        [MarshalAs(UnmanagedType.LPUTF8Str)] string subsystem,
        [MarshalAs(UnmanagedType.LPUTF8Str)] string category);

    [LibraryImport(Library)]
    internal static partial void os_log_impl(
        IntPtr dso, IntPtr log, byte type,
        [MarshalAs(UnmanagedType.LPUTF8Str)] string format,
        IntPtr buf, uint size);
}
```

Subsystem: `"pro.auracore.privhelper"`, categories: `"audit"`, `"xpc"`, `"lifecycle"`. Inspectable via `log show --predicate 'subsystem == "pro.auracore.privhelper"'`.

Fallback (when P/Invoke fails ‚Äî e.g. dev builds on Windows host): `Console.Error.WriteLine` with a `[PRIVHELPER] action=...` prefix. `launchd` plist's `StandardErrorPath` writes this to a rotating file.

Audit line format (both channels):
```
[PRIVHELPER audit] action=<id> caller_pid=<pid> caller_team=<team_id> argv=<joined> exit=<code> duration_ms=<n>
```

### 3.12 Testing strategy on Windows dev host

**Choice: Mirror Linux approach exactly.**

1. **Unit tests** ‚Äî mock `IXpcConnectionFactory` / `IXpcMessageHandler` interfaces. No real XPC wire touched. NSubstitute for doubles.
2. **Integration tests** ‚Äî `MacOSShellCommandService` routes through injectable factory; tests supply mocks.
3. **Real XPC validation** ‚Äî carried forward to manual macOS VM scope per Phase 5.2 spec ¬ß8.

Hermetic XPC test harness is explicitly rejected (as it was for D-Bus on Linux) ‚Äî infrastructure cost too high for first-ship.

## 5. Security summary table

| Layer | Threat | Mitigation |
|---|---|---|
| Transport | Rogue peer on Mach service | `SecCodeCheckValidity` on every call (¬ß3.3) |
| Wire | Deserialization gadgets | Primitive-typed `xpc_dictionary` only, strict key allowlist (¬ß3.2) |
| Argv | Command injection | Per-action regex validator + `ShellMeta` metachar rejection (¬ß3.9) |
| Path | SIP / arbitrary-file-write | SIP-prefix rejection at validation layer (¬ß3.10) |
| Helper identity | Stale helper after app update | `HelperVersion.Current` over XPC; client triggers re-install on mismatch (umbrella ¬ß3.2) |
| Code signing | Unsigned dev builds | `MacOSShellCommandService` fallback to `InProcessShellCommandService` + warning log (¬ß3.6) |
| Audit | Silent privileged ops | `os_log`/stderr with action/pid/team/argv/exit every call (¬ß3.11) |
| Supply chain | NuGet CVEs | Zero new third-party packages in 5.2.2; CVE-check every restore |

## 7. Appendix A ‚Äî Ship-prep commands (documentation only; not executed in 5.2.2)

```bash
# 1. Build universal helper binary
dotnet publish src/Service/AuraCore.PrivHelper.MacOS -r osx-x64   -c Release \
    -p:SelfContained=true -p:PublishSingleFile=true -o dist/mac-x64
dotnet publish src/Service/AuraCore.PrivHelper.MacOS -r osx-arm64 -c Release \
    -p:SelfContained=true -p:PublishSingleFile=true -o dist/mac-arm64
lipo -create dist/mac-x64/pro.auracore.privhelper dist/mac-arm64/pro.auracore.privhelper \
    -output dist/pro.auracore.privhelper

# 2. Substitute team ID placeholder
sed -i '' "s/AURACORE_TEAM_ID_PLACEHOLDER/$TEAM_ID/g" dist/pro.auracore.privhelper

# 3. Sign helper with hardened runtime
codesign --force --options runtime \
    --entitlements src/Service/AuraCore.PrivHelper.MacOS/assets/entitlements.plist \
    --sign "$CERT_ID" dist/pro.auracore.privhelper

# 4. Embed into app bundle + sign outer bundle
cp dist/pro.auracore.privhelper AuraCorePro.app/Contents/Library/LaunchDaemons/
cp src/Service/AuraCore.PrivHelper.MacOS/assets/pro.auracore.privhelper.plist \
    AuraCorePro.app/Contents/Library/LaunchDaemons/
codesign --force --options runtime --deep --sign "$CERT_ID" AuraCorePro.app

# 5. Notarize
xcrun notarytool submit AuraCorePro.app.zip \
    --apple-id "$APPLE_ID" --team-id "$TEAM_ID" --password "$APP_PASSWORD" --wait

# 6. Staple
xcrun stapler staple AuraCorePro.app
```
